nu.xom.xerces contains classes used to adapt the Xerces SAX parser for use with XOM. The primary effect is preventing the parser from accepting XML 1.1 documents. XOM is not designed to handle XML 1.1. This is a feature, not a bug. You do not need to use XML 1.1 and should not use XML 1.1. If you have any questions about this see the FAQ or Item 3 of Effective XML. Most users can safely ignore this package. The only reason you might need to refer to it is if you want to set some SAX features or properties on the parser before creating a Builder. In this case, using XML1_0Parser instead of org.apache.xerces.parsers.SAXParser will be somewhat more efficient, though it won't have any effect on the result.
