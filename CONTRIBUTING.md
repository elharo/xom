# Contributing to XOM

## Bug Reports

Bug reports are welcome through the [GitHub issue tracker](https://github.com/elharo/xom/issues).

When filing a bug, please include:
- A minimal, self-contained code example that reproduces the problem
- The version of XOM you are using
- The Java version and operating system you are running

## Feature Requests and Pull Requests

**Please be aware: XOM is a very stable, mature library. New features are only very occasionally accepted.**

XOM has been stable since version 1.0 and intentionally has a narrow scope. The vast majority of proposed changes and pull requests are not accepted. Before investing time in a pull request, please open an issue first to discuss whether your proposed change is likely to be accepted.

Changes most likely to be considered:
- Bug fixes with a clear test case demonstrating the defect
- Fixes for security vulnerabilities
- Improvements to documentation or error messages

Changes that are very unlikely to be accepted:
- New features or API additions
- Changes to existing behavior, even if arguably more correct
- Refactoring or code style changes
- Dependency upgrades unless required for security

If you do submit a pull request, please include a test that demonstrates the problem being fixed.
